# Pord — Persian Word

> یک ویرایشگر سند فارسیِ مدرن و کاملاً تحت وب، با تم سایت، فونت Vazirmatn، چیدمان RTL، افکت‌های گلس‌مورفیسم و خروجی PDF/Word با استایل جاسازی‌شده.

Pord (که برگرفته از **Persian Word** است) یک ویرایشگر بلوکی برای ساخت اسناد فارسیِ حرفه‌ای است که دقیقاً همان ظاهر و حس وب‌سایت شما را در خروجی PDF یا Word بازتولید می‌کند. هر چیزی که در ویرایشگر می‌بینید — فونت، رنگ‌ها، حاشیه‌ها، صفحه‌بندی A4 — دقیقاً همان چیزی است که در فایل نهایی دریافت می‌کنید.

## ویژگی‌های کلیدی

### ویرایش بلوکی با Drag-and-Drop
سند شما از «بلوک‌های» مستقل تشکیل شده — عنوان، زیرعنوان، پاراگراف، تصویر، جدول، کد، نقل‌قول و غیره. هر بلوک را می‌توانید به‌صورت مستقل جابجا، کپی، حذف، یا به نوع دیگر تبدیل کنید. رابط کاربری به کمک `@dnd-kit` امکان کشیدن و رها کردن لمسی و صفحه‌کلیدی را هم فراهم می‌کند.

### خروجی PDF و Word با استایل جاسازی‌شده
خروجی PDF با استفاده از `window.print()` روی یک iframe پنهان و قواعد `@page A4` ساخته می‌شود. خروجی Word یک HTML blob با فضای نام Microsoft Office است که در Word باز می‌شود. هر دو خروجی شامل موارد زیر به‌صورت جاسازی‌شده (Inline) هستند:

- فونت Vazirmatn (Regular/Medium/Bold) به‌صورت base64
- تمام استایل‌های CSS مرتبط
- تصاویر به‌صورت base64 (مستقل از منبع خارجی)

به این ترتیب فایل خروجی در هر سیستمی — آنلاین یا آفلاین — دقیقاً یکسان رندر می‌شود.

### پیست هوشمند با تشخیص خودکار نوع محتوا
وقتی متنی را از هر منبعی (ChatGPT، وب‌سایت، Word، ایمیل) کپی و در ویرایشگر پیست می‌کنید، Pord به‌طور خودکار ساختار را تشخیص می‌دهد:

- تیترها (H1/H2/H3) → بلوک title/h2/h3
- پاراگراف‌ها → بلوک paragraph
- لیست‌های نقطه‌ای و شماره‌دار → بلوک bullet
- نقل‌قول‌ها → بلوک quote
- بلوک‌های کد → بلوک code با زبان مشخص
- جداول HTML → بلوک table

### پیست تصویر با Ctrl+V
عکس‌های داخل کلیپ‌بورد را مستقیماً در ویرایشگر پیست کنید — به‌صورت خودکار به data URL تبدیل و به‌عنوان بلوک image درج می‌شوند. نیازی به آپلود روی سرور نیست.

### فهرست مطالب (TOC) خودکار
بلوک TOC سراسر سند را اسکن می‌کند و زیرعنوان‌ها، تیترهای بخش (h2) و تیترهای فرعی (h3) را شناسایی و با تورفتگی مناسب نمایش می‌دهد. می‌توانید انتخاب کنید کدام سطوح در فهرست بیایند. ورودی‌ها با شماره‌ی صفحه در خروجی PDF ظاهر می‌شوند (به کمک `target-counter`).

### لغت‌نامه‌ی انگلیسی خودکار
بلوک glossary تمام کلمات انگلیسی استفاده‌شده در سند را به‌طور خودکار شناسایی و به‌صورت مرتب‌شده فهرست می‌کند. می‌توانید برای هر کلمه معنی فارسی بنویسید؛ این معانی همراه سند ذخیره می‌شوند.

### پاورقی
با بلوک footnote یک نشانگر `[n]` در متن درج می‌کنید و متن پاورقی در انتهای سند لیست می‌شود. کلیک روی شماره به پاورقی و بالعکس پرش می‌کند.

### ذخیره‌ی سند و مدیریت چند سند
- **دکمه‌ی «ذخیره»** — سند فعلی را با نام دلخواه در `localStorage` ذخیره می‌کند
- **دکمه‌ی «سندهای من»** — فهرست تمام سندهای ذخیره‌شده با پیش‌نمایش، نام و تاریخ
- امکان بارگذاری، تغییر نام یا حذف هر سند

### خروجی و ایمپورت TXT (Round-trip)
- **خروجی TXT** — کل سند (meta + بلوک‌ها) را به یک فرمت markdown-style تبدیل و دانلود می‌کند
- **ایمپورت TXT** — فایل TXT را به بلوک‌های ساختاریافته بازمی‌گرداند (هم فرمت بومی Pord، هم متن خام و هم markdown ساده را تشخیص می‌دهد)
- این یعنی می‌توانید سند را به‌صورت متن ساده ذخیره کنید و بعداً بدون از دست رفتن هیچ اطلاعاتی بازگردانید

### Autosave دوره‌ای
هر ۶۰ ثانیه، سند فعلی به‌طور خودکار در `localStorage` ذخیره می‌شود. یک نوتیفیکیشن گوشه بالا چپ با انیمیشن pulse نشان می‌دهد که ذخیره انجام شده است. Autosave فوری (debounce شده) هم با هر تغییر فعال است.

### صفحه‌بندی ویرایشگر
برای جلوگیری از اسکرول بی‌نهایت، بلوک‌ها در صفحه‌های ۶تایی تقسیم می‌شوند. دکمه‌های «صفحه قبل» و «صفحه بعد» در بالای ویرایشگر ظاهر می‌شوند و نشان می‌دهند در کدام صفحه هستید.

### شمارش زنده‌ی کلمات
در پایین ویرایشگر، تعداد کلمات و کاراکترهای سند به‌صورت زنده نمایش داده می‌شود.

### جستجو در سند
جستجوی زنده در متن همه‌ی بلوک‌ها — موارد مطابق با هایلایت زرد نمایش داده می‌شوند.

## نصب و اجرای محلی

### ۱. دریافت کد

فایل ZIP را باز کنید:

```bash
unzip persian-doc-editor.zip
cd persian-doc-editor
```

یا اگر از Git استفاده می‌کنید:

```bash
git clone <repo-url> pord
cd pord
```

### ۲. نصب وابستگی‌ها

با bun (سریع‌تر، پیشنهادی):

```bash
bun install
```

یا با npm:

```bash
npm install
```

یا با yarn:

```bash
yarn install
```

### ۳. اجرای سرور توسعه

```bash
bun run dev
# یا
npm run dev
```

سپس مرورگر را روی [http://localhost:3000](http://localhost:3000) باز کنید.

### ۴. ساخت نسخه‌ی Production

```bash
bun run build
bun run start
```

سرور production روی پورت 3000 اجرا می‌شود (قابل تغییر با متغیر محیطی `PORT`).

## مدل داده

Pord از یک مدل ساده اما انعطاف‌پذیر استفاده می‌کند که همگی در `src/lib/doc-types.ts` تعریف شده‌اند:

### DocMeta

```typescript
interface DocMeta {
  title: string;
  author: string;
  date: string;
  showHeroSubtitle?: boolean;  // نمایش «ساخته‌شده با Pord» (پیش‌فرض: true)
  showFooterCredit?: boolean;  // نمایش «سازنده ارشی/Arshi» (پیش‌فرض: true)
}
```

### Block

هر بلوک یک `id` یکتا (UUID) و یک `type` دارد. سایر فیلدها بسته به نوع بلوک متفاوتند:

```typescript
type Block =
  | TextBlock       // title, subtitle, h2, h3, paragraph, quote, callout
  | BulletBlock     // { items: string[] }
  | ImageBlock      // { src, caption, alt, width, align }
  | DividerBlock
  | TableBlock      // { rows: string[][] }
  | CodeBlock       // { code, language }
  | SpacerBlock     // { height }
  | ColumnsBlock    // { columns: Block[][], columnCount }
  | FootnoteBlock   // { text }
  | TocBlock        // { title, includeSubtitle, includeH2, includeH3 }
  | GlossaryBlock   // { entries: GlossaryEntry[], autoDetect }
  | PageBreakBlock;
```

هر بلوک می‌تواند فیلدهای استایل اختیاری هم داشته باشد:
- `fontSize: "sm" | "md" | "lg" | "xl"`
- `blockWidth: "full" | "wide" | "medium" | "narrow"`
- `marginBottom: number`

## گردش کار خروجی

### PDF

1. `exportToPdf(meta, blocks)` صدا زده می‌شود.
2. `buildExportHtml()` یک HTML کامل می‌سازد با:
   - فونت Vazirmatn به‌صورت base64 در `@font-face`
   - تمام CSS در یک `<style>`_block
   - تصاویر به‌صورت base64
3. HTML در یک iframe پنهان load می‌شود.
4. `iframe.contentWindow.print()` صدا زده می‌شود.
5. مرورگر دیالوگ print را نمایش می‌دهد؛ کاربر می‌تواند به‌عنوان PDF ذخیره کند.

قواعد `@page` برای A4 و شماره‌ی صفحه در فوتر چاپ تنظیم شده‌اند:

```css
@page {
  size: A4;
  margin: 14mm 12mm 16mm 12mm;
  @bottom-center {
    content: "صفحه " counter(page) " از " counter(pages);
  }
}
```

### Word

1. `exportToWord(meta, blocks)` صدا زده می‌شود.
2. همان HTML ساخته می‌شود ولی با فضای نام MS Office اضافه‌شده در `<html>`.
3. HTML به‌صورت یک `Blob` با mime type `application/msword` در می‌آید.
4. یک لینک دانلود با پسوند `.doc` ایجاد و کلیک می‌شود.
5. فایل در Microsoft Word باز می‌شود و قابل ویرایش است.

### TXT (Round-trip)

سریالایز کردن:

```
§DOC-Pord v1
§META title=گزارش نمونه
§META author=تیم محتوا
§META date=2026-07-18
§META showHeroSubtitle=false
§BLOCKS
# گزارش نمونه
## بخش اول
این یک پاراگراف است.
- مورد اول
- مورد دوم
```python
print("hello")
```
| ستون ۱ | ستون ۲ |
| مقدار ۱ | مقدار ۲ |
```

پارس کردن: تابع `parseTxtToDocument()` این فرمت را به‌دقت بازمی‌گرداند. اگر فایل با `§DOC-Pord` شروع نشود، رفتار legacy به `parseTextToBlocks()` برمی‌گردد که markdown ساده را هم تشخیص می‌دهد.

## ذخیره‌سازی و Autosave

Pord از دو مکانیزم ذخیره‌سازی استفاده می‌کند:

### ۱. Autosave فوری (Debounce شده)
هر تغییر در `meta` یا `blocks`، بعد از ۶۰۰ms سکوت، در `localStorage` با کلید `doc-editor-state` ذخیره می‌شود. این state در بارگذاری مجدد صفحه بازمی‌گردد.

### ۲. Autosave دوره‌ای (هر ۶۰ ثانیه)
یک `setInterval` هر ۶۰ ثانیه همان state را دوباره ذخیره می‌کند و یک toast در گوشه بالا چپ با انیمیشن pulse نشان می‌دهد. این مکانیزم حتی اگر کاربر تغییر نکرده باشد هم اجرا می‌شود تا اطمینان حاصل شود state همیشه به‌روز است.

### ۳. Snapshots دستی
دکمه‌ی «ذخیره» یک snapshot با نام دلخواه در `localStorage` با کلید `pord-snapshots` ایجاد می‌کند. هر snapshot شامل:
- `id` (UUID)
- `name` (نام دلخواه کاربر)
- `meta` و `blocks`
- `createdAt` (timestamp)

می‌توانید چندین snapshot داشته باشید و هر کدام را مستقل بارگذاری، تغییر نام یا حذف کنید.

## صفحه‌بندی ویرایشگر

برای جلوگیری از اسکرول طولانی، ویرایشگر بلوک‌ها را در صفحه‌های ۶تایی نمایش می‌دهد:

- **«صفحه قبل»** و **«صفحه بعد»** در نوار بالای ویرایشگر
- نمایش «صفحه X از Y (بلوک‌های A تا B از N)»
- وقتی بلوک جدید insert می‌شود، خودکار به صفحه‌ی درست می‌پرد
- drag-and-drop فقط در همان صفحه کار می‌کند (برای جابجایی بین صفحه‌ها از دکمه‌های up/down استفاده کنید)
- یک divider در انتهای هر صفحه نشان می‌دهد کجای سند هستید

برای تغییر تعداد بلوک در هر صفحه، ثابت `BLOCKS_PER_PAGE` را در `src/app/page.tsx` تغییر دهید.

فایل‌های woff2 را در `public/fonts/` جایگزین کنید و `@font-face` را در `globals.css` و `src/lib/export-doc.ts` به‌روزرسانی کنید.

### تغییر اندازه‌ی صفحه

۱. نوع را به `BlockType` در `src/lib/doc-types.ts` اضافه کنید.

۵. در `serializeBlockToTxt` و `parseTextToBlocks` هم پشتیبانی اضافه کنید.

## عیب‌یابی

### خروجی PDF فونت درست ندارد
مطمئن شوید فایل‌های `Vazirmatn-*.woff2` در `public/fonts/` موجود هستند. در خروجی PDF، فونت به‌صورت base64 در HTML جاسازی می‌شود.

### فهرست مطالب (TOC) خالی است
۱. بلوک‌های `subtitle`، `h2` یا `h3` به سند اضافه کنید.
۲. روی بلوک TOC کلیک کنید و مطمئن شوید toggleهای سطح روشن هستند.
۳. در پنل ویرایش بلوک TOC، پیش‌نمایش زنده ورودی‌های شناسایی‌شده نمایش داده می‌شود.

### سند ذخیره‌شده در `localStorage` پاک شده
متأسفانه `localStorage` فقط در همان مرورگر و همان دامنه قابل دسترسی است. برای انتقال سند بین دستگاه‌ها از دکمه‌ی «خروجی txt» استفاده کنید.

### خروجی PDF صفحه‌بندی نمی‌شود
برخی مرورگرها در حالت headless صفحه‌بندی را پشتیبانی نمی‌کنند. از مرورگر کروم یا اجکس نسخه‌ی جدید استفاده کنید.

### خطای TypeScript در حین build
این پروژه شامل چند خطای TS از پیش موجود است (در `examples/` و `skills/`) که ارتباطی به کد اصلی ندارند. می‌توانید با `npx tsc --noEmit` فقط فایل‌های `src/` را بررسی کنید.

### پورت ۳۰۰۰ اشغال است
با دستور `PORT=3001 bun run dev` پورت را تغییر دهید.
